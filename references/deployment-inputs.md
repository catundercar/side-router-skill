# 部署输入要求

## 默认输入

规划部署模式下，用户至少应提供：

- 主路由 IP
- 旁路由目标 IP
- Ubuntu 版本
- LAN 网卡名
- 是否让客户端网关指向旁路由
- 是否让客户端 DNS 指向旁路由
- 是否启用 `AdGuardHome`
- 是否启用 `Mihomo/Clash Meta`
- 是否启用 `TProxy`
- 是否启用 `fake-ip`
- 节点来源：订阅或本地静态配置
- 指定服务域名的出口偏好：主策略组、备用策略组或区域偏好

## 建议补充

- 主路由 DHCP 是否可修改
- 当前是否已有旁路由或旧代理配置
- 是否需要兼容腾讯游戏、微信、国内视频平台
- 是否需要暴露 Mihomo 控制口
- 是否需要本地 Web UI

## 代执行额外输入

只有在用户希望 AI 直接代执行部署时，才需要这些信息：

- SSH 地址或可用终端
- 用户名
- 提权方式：`sudo` 或 `root`
- 是否允许重启服务
- 是否接受短时断网
- 是否有本地控制台兜底

## 默认不要求的信息

以下信息不应作为默认输入要求：

- SSH 密码
- 私钥全文
- 订阅完整 URL
- token
- API 密钥

如果需要生成真实生产配置，优先建议用户：

- 使用占位符
- 只提供必要片段
- 对订阅地址和 token 脱敏
