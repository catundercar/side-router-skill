# 离线恢复手册

## 原则

没有网络时，先恢复网络，再分析根因。

不要在全网断开的状态下继续盲改旁路由配置。优先恢复一台测试设备的联网能力，确保可以继续查资料、联系 AI、访问控制端口。

## 一台测试设备的最小恢复流程

1. 选一台测试设备，不要一上来改全网
2. 临时把测试设备默认网关改回主路由
3. 临时把测试设备 DNS 改回主路由或公共 DNS
4. 验证是否可以访问公网 IP 和域名
5. 恢复联网后，再采集旁路由诊断信息

## 判断方向

### 能 ping 公网 IP，不能打开域名

优先怀疑：

- DNS 配置错误
- AdGuardHome 上游故障
- Mihomo DNS 监听异常
- fake-ip 或 DNS 劫持异常

### 公网 IP 也不通

优先怀疑：

- 客户端网关错误
- 旁路由默认路由错误
- TProxy / 策略路由黑洞
- 旁路由服务未启动或网卡不通

## 恢复后应提供给 AI 的信息

```bash
ip a
ip route
ss -lntup
systemctl status clash --no-pager
systemctl status adguardhome --no-pager
curl -s http://127.0.0.1:9090/version
```

再补充：

- 测试设备当前网关
- 测试设备当前 DNS
- 最近一次改动内容
- 关键配置片段

## 全网断网时的安全建议

- 先恢复一台设备，不要立刻动主路由 DHCP
- 不要连续改 DHCP、DNS、Mihomo、iptables 四个面
- 如果没有本地控制台，避免重启关键服务
- 如果 SSH 依赖当前旁路由链路，优先恢复本地访问路径再操作
