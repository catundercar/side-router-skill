Deploy-first Network Policy Skill

Side Router Skill

一个面向 Ubuntu 旁路由部署的开源网络策略 skill。它优先解决部署规划、配置生成、上线检查和回滚设计， 同时覆盖 DNS、网关、TProxy、fake-ip、指定服务域名策略以及断网后的恢复指引。

软路由能做什么

  • 接管局域网设备默认网关，让所有流量先进入旁路由再做统一分流。
  • 接管 DNS，把 AdGuardHome -> Mihomo DNS 串成一条稳定的解析与过滤链路。
  • 用 TProxy + iptables + policy routing 对 TCP 流量做透明代理，减少终端逐台配置。
  • 给指定服务域名配置专用策略组，让办公、开发或外部 SaaS 的出口更稳定。
  • 让国内视频、微信、腾讯游戏、语音链路优先直连，降低 fake-ip 和误代理带来的兼容性问题。
  • 用 proxy-providers 管理订阅，把节点更新和主配置分离，避免规则被覆盖。

适合的网络

家庭网络

统一 DNS、分流和指定业务出口，不需要每台终端分别装代理。

小型办公网络

对开发、办公或外部 SaaS 工具做统一出口治理，同时保留国内服务直连。

高兼容性场景

需要同时兼顾视频、微信、腾讯游戏、语音链路和 AI 服务时更有价值。

高风险提示

  • 这个 skill 会直接涉及默认网关、DNS、iptables、TProxy 和关键 systemd 服务。
  • 让 AI 直接执行部署时，可能导致 SSH 断连、全网断网、DNS 全部失效。
  • 默认应该先用规划部署模式生成方案、验证清单和回滚步骤，再决定是否代执行。
  • 如果用户已经没网，先恢复一台测试设备的网关和 DNS，再继续排障。
  • `fake-ip` 不是中性设置，需要和 `DIRECT` 规则、AdGuardHome 放行规则一起设计。

适用场景

规划部署

生成部署步骤、配置模板、上线验证和回滚清单。

审查现有部署

检查网关接管、DNS 链路、TProxy 和服务域名策略是否合理。

断网恢复

先恢复一台设备联网，再采集诊断信息交给 AI 分析。

安装方式

一键安装

首选安装方式是通过 npx skills add 自动下发到本机已支持的 agent。

npx skills add catundercar/side-router-skill --skill side-router-skill -y

Claude Code

只安装到 Claude Code：

npx skills add catundercar/side-router-skill --skill side-router-skill -a claude-code -y

Cursor

只安装到 Cursor：

npx skills add catundercar/side-router-skill --skill side-router-skill -a cursor -y

Codex

只安装到 Codex：

npx skills add catundercar/side-router-skill --skill side-router-skill -a codex -y
# 同时安装到多个工具 npx skills add catundercar/side-router-skill --skill side-router-skill \ -a claude-code -a codex -a cursor -y

只有在不能使用 npx skills add 时，才建议手动复制仓库中的 SKILL.md、.claude/agents/ 或 .cursor/rules/。

默认输入

  • 主路由 IP
  • 旁路由目标 IP
  • Ubuntu 版本
  • LAN 网卡名
  • 是否接管网关与 DNS
  • 是否启用 AdGuardHome / Mihomo / TProxy / fake-ip
  • 节点来源方式与指定服务域名的出口偏好

代执行额外输入

  • SSH 地址或可用终端
  • 登录用户名
  • 提权方式：sudo 或 root
  • 是否接受短时断网
  • 是否有本地控制台或物理接入兜底
