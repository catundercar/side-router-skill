高风险提示

这是高风险网络改造 skill。它可能影响默认网关、DNS、透明代理和整段局域网的可用性。 默认应先用规划部署模式确认方案、检查项和回滚方案，再决定是否允许 AI 代执行。

  • 不要在没有本地控制台或物理接入能力时盲改 DHCP、DNS、iptables 或关键服务。
  • 如果已经断网，先恢复一台测试设备的网关和 DNS，再继续排障。
  • fake-ip 不是中性配置，它会直接影响视频、腾讯游戏语音、微信图片和部分 AI 子域名。

部署模式

规划部署模式

默认模式，不需要 SSH 登录信息。用于生成部署步骤、配置模板、systemd 规范、上线检查单和回滚步骤。

代执行部署模式

只有用户明确授权并提供登录方式、提权路径、断网容忍度和恢复通道时才适合启用。

最小输入

  • 主路由 IP
  • 旁路由目标 IP
  • Ubuntu 版本
  • LAN 网卡名
  • 是否接管网关与 DNS
  • 是否启用 AdGuardHome / Mihomo / TProxy / fake-ip
  • 节点来源方式与 AI 分流目标
